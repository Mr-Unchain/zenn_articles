## 開発はPCでやるものか？

基本はYESです。とはいえ、次みたいな状況だと「PCを開くまでが一番重い」ことがあります。

- PCを立ち上げる気力がない（でも作業は進めたい）
- アイデアが湧いた瞬間に形にしたい
- そもそもPCを持ち歩いていない

結論としては、**“条件がそろえば”スマホだけでも「小さく作って公開」までは十分いける**、という感触でした。

## 今回の前提（構成）

「スマホだけで完結」を成立させる鍵は、ローカル環境を捨ててクラウドに寄せることです。

- 対象：既存の技術ブログ（GitHubで管理）
- デプロイ：Vercel（プッシュ/マージで自動デプロイ）
- 実装：Codex（ChatGPT経由）
- 操作：iPhone（GitHubアプリ + ブラウザ）

ポイントは、**コードを書く手をスマホからAIに“委任”し、確認と意思決定だけスマホでやる**、という割り切りです。

## 実際にやったこと

### 1. ChatGPTに「Codex用の指示」を作らせる

スマホで長い指示を書くのはしんどいので、先にChatGPTへ相談して「Codexに渡すプロンプト」を作ってもらいます。

ここで意識すると良いのは、次の3点です。

- 目的：何を追加したいのか（例：ポートフォリオページ追加）
- 制約：既存の構成/デザイン方針/技術スタック
- 完了条件：どの状態になればOKか（画面・ルーティング・リンクなど）

### 2. Codexに渡す指示（プロンプト例）

実際にCodexへ渡す指示は、**“実装タスク”として読める文章**にすると通りが良いです。

```text
あなたはこのリポジトリのコントリビューターです。
目的: 技術ブログに「Portfolio」ページを追加し、トップ/ナビから遷移できるようにしてください。

要件:
- 既存のデザイン/コンポーネント/ルーティング規約に合わせる
- ページ内容は仮でよい（自己紹介、スキル、個人開発、連絡先のセクションを用意）
- レスポンシブ対応（スマホで崩れない）

作業手順:
1) まず既存のページ構成を確認して方針を箇条書きで提案
2) その方針で実装し、差分をPRとして作成
3) 変更点と確認観点を最後にまとめる
```

この形にしておくと、スマホ側では「方針OK/NG」「追加で直したい点」を短文で返すだけで回せます。

### 3. GitHub → Vercel の流れで確認する

スマホ作業が成立するのは、PRとPreviewデプロイがあるからです。

- CodexがPRを作る
- GitHubアプリで差分を見る（ざっくりでOK、細部は必要なところだけ）
- VercelのPreview URLで実際のページを開いて確認

UIの最終確認が“実物”でできるので、スマホでも判断がしやすいです。

## できたこと / きつかったこと

### できたこと

- 新規ページ追加（ポートフォリオ）
- スマホで表示崩れを確認
- PR → Preview → マージで本番反映

### きつかったこと

- 細かいデザイン調整（余白、微妙な整列、ピクセル単位の確認）
- 差分レビューの効率（大きいPRになるほどPCが恋しくなる）
- “考える”作業（情報整理や設計判断は結局PCが楽）

## このやり方が向いている場面 / 向いていない場面

### 向いている

- 新規ページ追加など、粒度が大きめで「まず動くもの」が欲しい
- 軽微な修正（文言、リンク、レイアウト微調整）
- 初動（アイデアを形にしてからPCに引き継ぐ）

### 向いていない

- 大規模リファクタ（影響範囲の把握がつらい）
- 複雑な状態管理やバグ修正（再現と調査が重い）
- 厳密なテスト/デバッグが必要な作業

## まとめ

スマホだけで開発を“完結”させるというより、**「実装はAI、検証と意思決定はスマホ」**に分業すると現実的になります。

次の2つがそろっているプロジェクトなら、特に相性が良いです。

- GitHubでのPR運用ができる
- VercelなどPreviewデプロイが自動で回る

「今すぐ動くものが欲しい」タイミングの選択肢として、かなりアリでした。
