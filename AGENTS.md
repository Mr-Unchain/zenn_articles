# Repository Guidelines

## プロジェクト構成とモジュール整理
このリポジトリは `zenn-cli` を使った Zenn 向けコンテンツ管理用です。
- `articles/`: 単体記事の Markdown ファイル。ファイル名はそのままスラッグになります（例: `my-first-post.md`）。
- `books/`: Zenn で管理する複数チャプター形式の書籍コンテンツ。
- `package.json`: 依存関係とスクリプト定義。
- `CLAUDE.md`: 既存のエージェント向け運用メモ。

記事ごとの補助資料は関連コンテンツの近くに置き、不要なツール設定ファイルは追加しないでください。

## ビルド・テスト・開発コマンド
- `npm install`: `zenn-cli` と lockfile 固定の依存関係をインストールします。
- `npx zenn preview`: ローカルプレビューサーバーを起動し、表示を確認します。
- `npx zenn new:article`: 新しい記事テンプレートを作成します。
- `npx zenn new:book`: 新しい書籍テンプレートを作成します。
- `npm test`: 現在はプレースホルダーで失敗するため、検証には使わないでください。

## コーディングスタイルと命名規則
- コンテンツは YAML フロントマター付き Markdown で記述します。
- 見出し（`##`, `###`）を使い、簡潔で読みやすい技術文書を心がけます。
- 記事ファイル名は小文字の kebab-case スラッグにします（例: `rust-error-handling.md`）。
- フロントマターは Zenn の慣例に従ってください: `title`, `emoji`, `type`, `topics`, `published`。

## テスト方針
このリポジトリには自動テストはありません。
- PR 作成前に `npx zenn preview` で表示確認を行ってください。
- フロントマター項目、リンク切れ、デスクトップ/モバイル幅での表示を確認してください。
- 変更量が多い場合は、最低 1 回は全体を通してローカルプレビューを確認してください。

## コミットとプルリクエストの指針
直近のコミットは、1 変更ごとに短く具体的なメッセージ（日本語中心）で記述されています。
- コミットは 1 つの論理的変更単位に絞ってください。
- 件名は具体的にしてください（例: `記事タイトルを修正`、`Update zenn-cli to 0.2.11`）。
- PR には目的、変更したパス（例: `articles/...`）、プレビュー確認内容を記載してください。
- レイアウトやレンダリング結果が大きく変わる場合はスクリーンショットを添付してください。

## セキュリティと設定の注意
- シークレット、トークン、個人環境ファイルはコミットしないでください。
- 依存関係を更新した場合は、再現性維持のため `package-lock.json` も合わせてコミットしてください。
